# Strategy — Real Estate CRM SaaS Landing Page

English strategy notes; all customer-facing copy examples are Turkish.

## 1. Positioning

Position as a **vertical CRM built for Turkish real estate offices**, not a generic
CRM adapted to real estate. The core promise is operational: portfolio, customer,
request, agent performance, pipeline and commission in one panel. The enemy is not
"competing CRMs" — it is **Excel + WhatsApp + notebooks**. Every section attacks
that status quo.

Differentiators vs generic CRM tools:

1. Real-estate-native objects (portföy, talep, yer gösterme, komisyon) instead of
   generic "deals" and "contacts".
2. Request↔portfolio matching — the feature Excel can never do.
3. Broker-first reporting: agent-level performance is a first-class view.
4. Commission/accounting tracking including shared (co-brokerage) sales.
5. Turkish-language product, Turkish onboarding, sector vocabulary.
6. Optional marketing services (Google/Meta Ads, SEO, site) — CRM + demand
   generation from one partner, which no horizontal CRM offers.

## 2. Ideal customer profile

- Broker/owner of a 3–15 agent office in a metropolitan or coastal market;
  currently on Excel/WhatsApp; feels loss of control as the team grows.
- Secondary: individual consultants professionalizing; franchise structures
  needing multi-office visibility; construction-company sales teams.
- Buying trigger: a lost deal caused by a forgotten request, or an agent leaving
  with "their" portfolio in a personal notebook.

## 3. Package logic

Three tiers segmented by team size and control needs (Başlangıç 1–3 users,
Profesyonel 5–15, Kurumsal multi-office). No public prices at MVP stage —
"demo sonrası tekliflendirilir" keeps pricing flexible and forces the demo call,
which is the actual conversion event. Introduce public pricing only after
5–10 paid references establish willingness-to-pay.

## 4. Single page vs multi-page

**Single page for MVP.** Ads traffic converts best on one focused page with one
action (demo request); multi-page dilutes budget and adds maintenance. The
architecture (sections as standalone components, copy as data) converts to
multi-page without rework when SEO content becomes a priority.

## 5. Form handling decision

**Internal API route + Resend** chosen because: zero client-side third-party
dependency, lead data stays under our control, trivially swapped for a Supabase
`leads` insert when the CRM lead module is ready (same endpoint contract), and
works with no configuration at all (log fallback) so deployment is never blocked.
Formspree remains a 1-line switch via `NEXT_PUBLIC_DEMO_FORM_ENDPOINT` if a
no-code option is ever preferred.

## 7. Funnel

Ad traffic → landing page → demo form → first contact (same day) → live demo →
pilot usage (2–4 weeks) → paid subscription → training/onboarding → upsell
professional services (data migration, ads, SEO, site).

Measurement: GA4 event on form success (add `gtag('event','generate_lead')` /
`fbq('track','Lead')` in `DemoForm` success branch once IDs exist), GTM for the
rest. Pixel + GA placeholders are already wired in `layout.tsx`.

1. Ücretsiz demo talep edin.
2. Ofisinizi tek panelde görün.
3. 2 dakikada formu doldurun, demoyu planlayalım.
4. Paketleri inceleyin, teklif alın.

Audience suggestions: interest targeting (real estate, emlak, gayrimenkul

## 11. Risks / open points

- No real testimonials yet → trust section is honest by design; replace with
  pilot-customer quotes as soon as they exist (with permission).
- No public pricing → some visitors bounce; acceptable while demo-led.
- KVKK: before scale, add a proper Gizlilik Politikası and Açık Rıza text to the
  form (footer links are placeholders).
- OG image missing (add `public/og-image.png` 1200×630).
- Feature claims mirror the actual CRM's implemented modules; keep
  `content/landing.ts` in sync with the product to avoid overpromising.
- Font loading is runtime (Google Fonts); self-host via `next/font` before
  performance-tuning for Core Web Vitals.

3. Add OG image + favicon.

6. Launch 1 Google Search campaign (exact-match emlak crm cluster) + 1 Meta
   retargeting-ready campaign with the copy above.
7. After first 10 demos: iterate headline/pricing section from objections heard.
